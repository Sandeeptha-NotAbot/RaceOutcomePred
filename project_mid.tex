\begin{abstract}
Our task in this project is to predict whether a Formula 1 driver will finish on the podium (top 3) in a given race before it begins. Our motivation is a strong interest in the sport and in connecting machine learning theory to real-world structured data. We use data from the 2014--2024 hybrid-turbo era, engineering pre-race features such as grid position, qualifying result, and rolling driver and constructor performance statistics. We compare five models: Logistic Regression (L1 and L2), Support Vector Machine (linear and RBF kernels), and Decision Tree. Our best-performing model, SVM Linear, achieves a ROC-AUC of 0.946 on the 2022 validation set and 0.930 on the 2023--2024 test set.
\end{abstract}

\section{Introduction}

Formula 1 is the highest level of international auto racing, featuring 24 racing weekends across international circuits. This means there will be a different track to race on each event weekend. Different teams are racing as well, each with two racers. Because vehicle performance varies significantly between teams and regulations change over time, predicting race outcomes is a challenging but tractable machine learning problem.
 
This project uses the Formula 1 World Championship dataset from Kaggle, scoped to the 2014--2024 hybrid-turbo era. Pre-2014 data is excluded because major regulation changes make constructor and driver strength non-comparable across eras. Similarly, 2026 data is excluded due to the most significant regulation overhaul in the sport's history, which reshuffled the entire power unit landscape. The prediction task is binary classification: given pre-race features for a driver entering a race, predict whether they will finish on the podium (positions 1--3).

The models evaluated are Logistic Regression (L1 and L2 regularization), Support Vector Machine (linear and RBF kernels), and Decision Tree. Features include starting grid position, qualifying classification position, driver and constructor podium rates for the current season, driver average grid position for the current season, and a teammate podium rate differential.

\section{Related Work}

There has been a similar study to ours by Spencer Staub at The George Washington University in Autumn 2022, titled "Formula One Race Prediction Model." He took a look at all the features in our data, such as drivers' previous races at tracks, teams' past results, qualifying win rates, vehicle engineering, and drivers' past finishes. They used similar models we used in Logistic Regression and SVM, plus one we did not use: XGBoost. Spencer also took a look at the improvement results from the previous season, which had an accuracy of 77 percent, suggesting the stats would be in our favor and something to take into account when comparing our work with past stats. In their future work, they plan to use lap data for testing, which we currently have in our project, to help with predicting the outcomes of future races.

\section{Methods}

\subsection{Dataset Collection and Era Filtering}
 
The dataset is sourced from the Formula 1 World Championship dataset on Kaggle, which includes race results, qualifying times, driver metadata, constructor metadata, and circuit information dating back to 1950. We restrict our analysis to 2014--2024 (the hybrid-turbo era) for two reasons. First, the 2014 regulation change introduced a fundamentally different power unit architecture, making pre-2014 constructor strength non-transferable. Second, limiting to a single regulatory era ensures that the features we engineer, particularly constructor and driver rolling statistics that reflect comparable competitive conditions. After filtering, the dataset contains 228 races and 4,626 driver-race entries, with a podium rate of 14.8\% (approximately 3 podiums per race across $\sim$20 drivers).
 
\subsection{Feature Engineering}
 
All features are constructed using only information available before a race starts to ensure no data leakage. Rolling statistics use a \texttt{shift(1)} expanding window so that the current race is never included in its own feature calculation. The six features used are:
 
\begin{itemize}
    \item \textbf{Grid position:} The driver's starting position on the grid. Pit lane starts (recorded as 0) are replaced with 20 to reflect a back-of-grid disadvantage.
    \item \textbf{Qualifying position:} The driver's classification position from the qualifying session. Missing values are filled with grid position as a fallback.
    \item \textbf{Driver season podium rate:} The fraction of races in the current season (prior to this round) in which the driver finished on the podium.
    \item \textbf{Constructor season podium rate:} The fraction of race entries in the current season (prior to this round) in which the constructor's car finished on the podium.
    \item \textbf{Driver season average grid:} The driver's mean starting grid position across all prior races in the current season.
    \item \textbf{Teammate podium rate differential:} The driver's season podium rate minus their teammate's season podium rate, capturing relative within-team performance.
\end{itemize}
 
For the first race of each season, where no prior data exists, rolling statistics are initialized to the season mean to avoid propagating null values into the model.
 
\subsection{Label}
 
The binary label is derived from \texttt{positionOrder}: a value of 1 indicates a podium finish (positions 1--3), and 0 indicates a non-podium finish. We use \texttt{positionOrder} rather than \texttt{position} because \texttt{position} is null for drivers who retire from the race (DNF), while \texttt{positionOrder} always contains a value.
 
\subsection{Class Imbalance}
 
Since only 14.8\% of entries are podium finishes, a naive model could achieve $\sim$85\% accuracy by predicting ``no podium'' for every entry. To address this, all models use \texttt{class\_weight=`balanced'}, which adjusts class weights inversely proportional to their frequency.
 
\subsection{Train / Validation / Test Split}
 
To avoid data leakage from future races into the training set, we use a temporal split rather than random shuffling:
 
\begin{itemize}
    \item \textbf{Train:} 2014--2021 (3,267 examples)
    \item \textbf{Validation:} 2022 (440 examples)
    \item \textbf{Test:} 2023--2024 (919 examples)
\end{itemize}
 
A random split would allow the model to train on, for example, 2023 race data while being tested on 2022 race data — inflating performance metrics without reflecting real predictive ability.
 
\subsection{Models}
 
All models are implemented as \texttt{sklearn} pipelines combining a \texttt{StandardScaler} with the classifier. Scaling is essential for Logistic Regression and SVM, which are sensitive to feature magnitude.
 
\begin{itemize}
    \item \textbf{Logistic Regression (L2):} Linear baseline classifier with L2 regularization. Provides interpretable coefficients indicating feature importance.
    \item \textbf{Logistic Regression (L1):} Variant with L1 regularization, which drives some coefficients to zero, enabling implicit feature selection.
    \item \textbf{SVM Linear:} Support vector machine with a linear kernel. Finds the maximum-margin hyperplane separating podium from non-podium entries.
    \item \textbf{SVM RBF:} Support vector machine with a radial basis function kernel, capable of modeling nonlinear decision boundaries.
    \item \textbf{Decision Tree:} Nonlinear, interpretable model. Maximum depth is capped at 5 and minimum samples per leaf at 20 to reduce overfitting.
\end{itemize}
 
\subsection{Evaluation Metrics}
 
Given the class imbalance, accuracy alone is insufficient. We report five metrics:
 
\begin{itemize}
    \item \textbf{Accuracy:} Fraction of all predictions that are correct.
    \item \textbf{Precision:} Of all entries predicted as podium, the fraction that actually finished on the podium.
    \item \textbf{Recall:} Of all actual podium finishes, the fraction the model correctly identified.
    \item \textbf{F1 Score:} Harmonic mean of precision and recall.
    \item \textbf{ROC-AUC:} Area under the receiver operating characteristic curve, measuring the model's ability to rank podium finishers above non-podium finishers regardless of threshold.
\end{itemize}

SVM Linear achieves the best overall performance on both sets, leading in F1 (0.707) and ROC-AUC (0.946) on validation. All models exhibit high recall (0.93--0.98), meaning they successfully identify nearly every real podium finish. Precision is lower across all models ($\sim$0.47--0.55), reflecting the difficulty introduced by the 14.8\% class imbalance, only 3 of approximately 20 drivers podium per race.
 
The small but consistent performance drop from validation to test (e.g., SVM Linear ROC-AUC: 0.946 $\rightarrow$ 0.930) is expected and reflects the growing dominance of \textbf{Max Verstappen and Red Bull Racing} in 2023--2024, which represents a slight distribution shift from the training period.
 
Feature importance analysis from the Decision Tree and Logistic Regression (L2) models indicates that qualifying position and grid position are the strongest predictors of podium probability, consistent with our expectations. Constructor season podium rate also carries significant weight, supporting the hypothesis that constructor dominance is a key driver of race outcomes. The teammate podium rate differential was identified as the least important feature by both models.
 
A feature correlation heatmap (Figure \ref{fig:heatmap}) computed on the training set confirms that grid position and qualifying position are highly correlated with each other (as expected) and moderately correlated with the podium label. Driver and constructor season podium rates show moderate positive correlation with the label, while the teammate differential shows the weakest correlation.

\section{Future plan}

A key next step is to evolve this project from a solid classification model into a richer and more realistic race-prediction system. While the current model performs well using only pre-race features, future efforts should focus on enhancing generalization and practical utility.
 
One direction is implementing season-by-season cross-validation to ensure reliability across different regulatory periods. Expanding the feature set to include circuit type (street circuit vs. permanent track), recent race form, and reliability indicators would provide a more comprehensive representation of race conditions.
 
Since the model currently prioritizes high recall at the expense of precision, refining probability thresholds or adopting a ranking-based approach, ordering drivers by podium likelihood which would make predictions more actionable. Incorporating ensemble methods such as Random Forest or XGBoost could further improve performance. Finally, hyperparameter tuning via grid search on regularization strength (C), kernel parameters, and tree depth is planned for the final report.
